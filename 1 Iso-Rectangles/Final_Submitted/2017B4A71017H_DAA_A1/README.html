Assignment Instructions

DAA Assignment 1

This project implements an optimal divide-and-conquer algorithm to compute Measure and Contour for a set of Iso-Rectangles, i.e. rectangles whose sides are alligned with the coordinate axes.

Problem Statement

We consider two geometrical problems that have been solved previously by line-sweep algorithms: the measure problem and the contour problem. Both problems involve determining some property of the union of a set of rectangles, namely the size and the contour (boundary) of the union. A single time-optimal divide-and-conquer algorithm is implemented to solve both problems. The divide-and-conquer algorithm is obtained by using a new technique called "separational representation", which extends the applicability of divide-and-conquer to orthogonal planar objects.

  • The Measure Problem: Given a set of n iso-oriented rectangles in the plane, determine the measure of their union. In other words we ask for the size of the covered area of the plane. 
  • The Contour Problem: For a set of iso-oriented rectangles in the plane, the contour is the boundary between the free and the covered area of the plane. In general it is a collection of disjoint "contour cycles", each of which is a sequence of alternating horizontal and vertical contour pieces.

Algorithm Discussion

Both the measure and the contour are properties of the union of a set of rectangles. So, we calculate a common parameter for both the problems, that is called stripes. The approach has two steps.

  • Compute the description stripes of a set of rectangles using the divide-and conquer algorithm. 
  • Given stripes, solve the specific problem for measure and contour.  Since the second step is relatively simple for both problems, it is essentially the one powerful algorithm stripes that solves both problems.

We now discuss the divide step in the divide and conquer algorithm. Let us define frame by the x interval which contains all the rectangles. This frame is [-∞, ∞] initially. Now at each divide step, we divide the frame into two halves based on the median of the x-coordinate of the vertical edges of the rectangles it contains, so that both halves contains almost equal number of vertical edges.
Now the base case is when there is exactly one vertical edge in the frame. For this case we calculate L, R, P and S. These are the four parameters that are used while merging the frame (in the conquer step).

  • L contains those left edges of frame whose partner is not in frame.
  • R is symmetric to L (for right edges).
  • P contains the y-projections of all endpoints of edges in the frame
  • S is the set of Stripes that will be required to calculate measure and contour

The next step is the merge step in which we merge L1, R1, P1, S1 (left frame) and L2, R2, P2, S2 (right frame) to obtain L, R, P and S for the merged frame. The merge step is done in 3 small steps of copy, blacken and concat.
Copy: Different y-partitions of left frame Stripes (S1) and right frame Stripes (S2) are made equal.
Blacken: After copying, S1 and S2 are blackened, meaning their x union fields are updated to indicate that they are completely covered by a rectangle.
Concat: Every two adjacent stripes S1 and S2 (with the same y-interval) are concatenated to form the stripes S of the merged frame.
Now, we have the required data structure Stripes for the entire frame.

  • Measure Problem: Now we can simply add (the length of x union * width of y interval) of each stripe to obtain the total measure.
  • Contour Problem: For finding contour we store x union of each stripe as a BST and we can get the horizontal contour edges by searching in the BST to answer a free subinterval query. After we have all the horizontal contour edges, we have the complete information and vertical edges can be calculated simply.

Issues Faced

Implementing the algorithm presented in the paper was a challenging task. Some of the major issues faced while coding are following:

  • Implementation: Since the problem has a lot of steps, the implementation is quite heavy. This issue was solved by planning the code well before actually writing it and designing all the classes and structures used properly.
  • Debugging: This was the most difficult step in the entire assignment. Once the code is finished, it would never be fully correct and requires debugging. Since there are a lot of small steps and many functions, it was really challenging to find where exactly the bug is. We overcame this issue by printing intermediate outputs of each functions and checking which intermediate output is not correct. So, we can find which function has the bug and hence, debug it.
  • Time Complexity: Some of the functions if implemented naively would be O(n²). So, it required some logical thinking to get the desired time complexity.
