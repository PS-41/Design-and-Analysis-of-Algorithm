Problem Statement

DAA Assignment 2

This project implements the Segmented Least Squares Algorithm to solve the line fitting problem using dynamic programming. It finds multiple best fit lines in the optimal partition using the cost multiplier for segments denoted by C.

Problem Statement

In this assignment, we consider the line fitting problem of finding multiple best fit lines given a set P of n points in 2-D plane. 
We are given a set P of n points in the pane, denoted by (x1, y1), (x2, y2), (x3, y3), . . . , (xn, yn) and suppose x1 < x2 < x3 < . . . < xn.
We have to partition P into some segments.
Segments is a subset of P that represents contiguous set of x-coordinates, denoted by S.
We have to compute the line minimizing the error with respect to the points in S.

Algorithm Discussion

We want to penalize too many partitions as well as the error.
So penalty is defined as the sum of 

  • The number of segments into which we partition P, times a fixed, given multiplier C > 0. 
  • For each segment, the error value of the best fit line through the segment.

So our objective is to find a partition with minimum penalty. We form a recurrence relation and use dynamic programming to solve the problem. Let OPT(i) be the optimal value for the points  p1,p2,...,pi.
Let e[i][j] denote the minimum error of any line through pi, pi+1,...,pj.
So, we want to compute OPT(n). 
Now if the last segment of the optimal partition is pi, pi+1,..., pn, then the value of optimal solution is 
OPT(n) = e[i][n] + C + OPT(i - 1) 
Since i can take only j distinct values, 
OPT(j) = min (e[i][j] + C + OPT(i - 1)) for 1 <= i <= j. 
Hence we have our dynamic programming solution.

Analysis based on Penalty Value

As we can see in the output shown above for different penalty values for a given input, the number of segments decreases on increasing the value of C. This is because C denotes the penalty for adding an extra segment. So for a very small value of C, we observe that every two consecutive points form a segment because the error for such segment = 0 and penalty cost is very low. Now, as we increase the value of C, the number of segments decreases to reduce the total penalty. For a very high value of C, we get exactly 1 segment as partitioning into another segment will incur a high penalty cost and hence, not optimal. 
The plot beside shows the dependence of number of segments on the penalty value (test case 2 and 3).

Another thing that we observe from the results shown above is that the penalty value has no effect on the space and time complexity of the algorithm. This is because the role of penalty value is only for calculating the error, and does not affect the complexity of our dynamic programming based algorithm. All the calculations involving the penalty value are O(1).

Issues Faced

Implementing the algorithm was a challenging task. Some of the major issues faced while coding are following:

  • Implementation: Since the problem has a lot of steps, the implementation is quite heavy. This issue was solved by planning the code well before actually writing it and designing all the classes and structures used properly.
  • Debugging: This was the most difficult step in the entire assignment. Once the code is finished, it would never be fully correct and requires debugging. Since there are a lot of small steps and many functions, it was really challenging to find where exactly the bug is. We overcame this issue by printing intermediate outputs of each functions and checking which intermediate output is not correct. So, we can find which function has the bug and hence, debug it.
